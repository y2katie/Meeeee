Project Overview

A customized learning experience created for GitHub's team, focused on accessibility best practices. The course emphasized real-world scenarios and interactive components to engage learners and build practical, lasting skills.

Role

UX Designer & Developer

Duration

3 months

Team

GitHub Team, Deque Systems

Tools

Figma, HTML/CSS, JavaScript

The Challenge

GitHub wanted to create an engaging way to train their team on accessibility best practices. Traditional training methods weren't resonating with developers, and the team needed a more interactive approach that would make accessibility concepts memorable and actionable.

The Solution

We designed an escape room experience where participants had to solve accessibility challenges to progress. Each room focused on different aspects of web accessibility, from keyboard navigation to screen reader compatibility. The gamified approach made learning engaging while reinforcing real-world skills.

Key Features

  • Interactive coding challenges based on real GitHub scenarios
  • Progressive difficulty levels to build confidence
  • Real-time feedback and hints system
  • Team collaboration features for group problem-solving
  • Comprehensive progress tracking and certificates

Impact

The escape room training resulted in a 85% completion rate compared to 40% for traditional training methods. Post-training assessments showed a 70% improvement in accessibility implementation across GitHub's development teams.
